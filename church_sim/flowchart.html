Boot Microcode Sequence

Boot Microcode (6 Steps, M-Elevation Active)

0Verify all CRs are zero (boot invariant) → FAULT if violated
1M-Elevation ON — enable trusted setup
2CR15 ← Namespace root (NS[0], no permissions)
3CR8 ← Boot Thread identity (NS[2])
4CR6 ← Boot C-List (NS[0], L+S via M gate)
5CR7 ← Boot CLOOMC (NS[1], executable code)
6M-Elevation OFF — boot complete → READY
Instruction Fetch & Condition Check

Common Prefix (Every Instruction)

1Fetch instruction word at PC
2If word = 0 → HALT (zero instruction)
3Decode: opcode[5] | cond[4] | dst[4] | src[4] | imm[15]
4Evaluate ARM condition (16 codes: EQ/NE/CS/CC/MI/PL/VS/VC/HI/LS/GE/LT/GT/LE/AL/NV)
5If condition false → skip, PC++ (conditional execution)
6Dispatch to instruction handler by opcode
7Post-execute: DR0 ← 0 (always cleared), emit state change
Security Gates — Trusted Security Base (TSB)

mLoad — Read Gate

1Parse GT: Version(7) | Index(17) | Perms(6) | Type(2)
2Bounds check: index < nsCount → BOUNDS fault
3Read namespace entry (word0: location, word1: B|F|G|limit, word2: version|seal)
4Update capability indicators (perms, B, F, version, seal)
5Version match: GT.ver == NS.ver → VERSION fault
6FNV seal validation → SEAL fault
7Permission check: GT has required perm (unless M-elevated) → PERMISSION fault
8Mark entry LIVE (set G-bit for GC)

mSave — Write Gate

1Parse source GT: Version(7) | Index(17) | Perms(6) | Type(2)
2Read source namespace entry → BOUNDS fault if null
3Update capability indicators (source GT perms, B, F, version, seal)
4Version match: source GT.ver == NS.ver → VERSION fault
5FNV seal validation on source → SEAL fault
6B-bit check: B=1 required (unless M-elevated) → BIND fault
7F-bit check: target F=1 → FAR object (requires HTTP/tunnel) → FAR fault if local-only
Called by: SAVE instruction (validates source GT before c-list write)
Symmetric counterpart to mLoad — together they form the complete TSB
Church Domain — 10 Instructions

0 — LOAD CRd, CRs, imm

1CR[src] = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L') — validate c-list, require L perm
3Bounds check: imm < nsCount, entry valid → BOUNDS fault
4FNV seal on target entry → SEAL fault
5Read GT word from entry's memory location
6Write to CR[dst] (occupied check) → CR_OCCUPIED fault
7PC++

1 — SAVE CRd, CRs, imm

1CR[dst] = source GT. Null? → NULL_CAP fault
2mSave(srcGT, targetIdx) — validate source: ver, seal, B=1, detect FAR
3CR[src] = c-list GT. Null? → NULL_CAP fault
4mLoad(clistGT, 'S') — validate c-list, require S perm
5If target slot invalid → create dynamic entry
6Register target in nsClistMap (GC reachability)
7Write source GT to target slot's memory location
8PC++

2 — CALL CRd

1CR[dst] = target GT. Null? → NULL_CAP fault
2mLoad(targetGT, 'E') — require E (entry) perm
3If handler registered (e.g., GC) → dispatch to safe Turing abstraction
4Push call frame: returnPC, all CRs, all DRs, flags
5Auto-clear B on all active CRs — callee cannot bind without explicit TPERM
6PC++

3 — RETURN CRd

1Call stack empty? → HALT (stack empty)
2Pop frame: restore returnPC
3PC ← returnPC

4 — CHANGE CRd, imm

1CR[src] = source GT. Null? → NULL_CAP fault
2Bounds check: imm < nsCount → BOUNDS fault
3Read entry at index imm → BOUNDS fault if null
4Read GT from entry, write to CR[dst] → CR_OCCUPIED fault
5PC++ (re-aims register without mLoad — index-based)

5 — SWITCH CRs, imm

1CR[src] must be non-null → NULL_CAP fault
2Swap CR[src] ↔ CR[imm&7] (atomic register swap)
3PC++

6 — TPERM CRd, preset[+B]

1Decode: presetCode = imm & 0xF, bSet = imm bit 4
2Reserved preset? → TPERM_RSV fault
3GT null? → Z=0, N=1 (fail flags), PC++
4Test: GT has all required permission bits?
5If B modifier + test passed → set B=1 on namespace entry word1
6Set flags: Z=pass, N=!pass, C=0, V=0
7PC++
Presets: CLEAR(0) R(1) RW(2) X(3) RX(4) RWX(5) L(6) S(7) E(8) LS(9) LE(10) SE(11) LSE(12) RWXLSE(13) + B flag(bit4)

7 — LAMBDA CRd

1CR[dst] = target GT. Null? → NULL_CAP fault
2mLoad(targetGT, 'X') — require X (execute) perm
3In-scope code application (no context save, lightweight)
4PC++

8 — ELOADCALL CRd, CRs, imm (Fused)

1CR[src] = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L') — LOAD phase: validate c-list
3Bounds + seal on target entry
4Load GT → CR[dst]
5mLoad(loadedGT, 'E') — TPERM+CALL phase: check E perm
6Push call frame (CRs, DRs, flags, returnPC)
7Auto-clear B on all active CRs
8PC++ (3 ops in 1 cycle: LOAD + TPERM(E) + CALL)

9 — XLOADLAMBDA CRd, CRs, imm (Fused)

1CR[src] = c-list GT. Null? → NULL_CAP fault
2mLoad(clistGT, 'L') — LOAD phase: validate c-list
3Bounds + seal on target entry
4Load GT → CR[dst]
5Check X permission on loaded GT → PERMISSION fault
6In-scope application (no context save)
7PC++ (3 ops in 1 cycle: LOAD + TPERM(X) + LAMBDA)
Turing Domain — 10 Instructions (inside safe abstractions)

10 — DREAD DRd, CRs, imm

1CR[src] = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'R') — require R perm
3Bounds check: imm ≤ entry limit → BOUNDS fault
4DR[dst] ← memory[location + imm]
5PC++

11 — DWRITE DRd, CRs, imm

1CR[src] = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'W') — require W perm
3Bounds check: imm ≤ entry limit → BOUNDS fault
4memory[location + imm] ← DR[dst]
5PC++

12 — BFEXT DRd, CRs, imm

1CR[src] = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'R') — require R perm
3Decode: pos = imm[9:5], width = imm[4:0]
4Validate pos+width ≤ 32 → BOUNDS fault
5DR[dst] ← (word >> pos) & mask(width)
6PC++

13 — BFINS DRd, CRs, imm

1CR[src] = data GT. Null? → NULL_CAP fault
2mLoad(dataGT, 'W') — require W perm
3Decode: pos = imm[9:5], width = imm[4:0]
4Validate pos+width ≤ 32 → BOUNDS fault
5word = (old & ~mask) | (DR[dst] << pos) & mask
6PC++

14 — MCMP DRd, DRs

1a = DR[dst], b = DR[src]
2Compute a − b (unsigned), set flags NZCV
3PC++ (no gate — pure arithmetic)

15 — IADD DRd, DRs, DRimm

1a = DR[src], b = DR[imm&0xF]
2DR[dst] = (a + b) mod 2³²
3Set flags: Z, N, C (carry), V (overflow)
4PC++ (no gate — pure arithmetic)

16 — ISUB DRd, DRs, DRimm

1a = DR[src], b = DR[imm&0xF]
2DR[dst] = (a − b) mod 2³²
3Set flags: Z, N, C (borrow), V (overflow)
4PC++ (no gate — pure arithmetic)

17 — BRANCH imm (signed offset)

1Sign-extend imm[15] to signed offset
2target = PC + offset. Range check → BOUNDS fault
3PC ← target (combine with cond for conditional branch)

18 — SHL DRd, DRs, imm

1shamt = imm & 0x1F (shift amount 0–31)
2DR[dst] = DR[src] << shamt
3Set Z, N, C (last bit shifted out)
4PC++ (no gate — pure arithmetic)

19 — SHR DRd, DRs, imm

1shamt = imm & 0x1F (shift amount 0–31)
2DR[dst] = DR[src] >>> shamt (logical shift)
3Set Z, N, C (last bit shifted out)
4PC++ (no gate — pure arithmetic)
PP250 Deterministic Garbage Collection

GC — Safe Turing Abstraction (via CALL)

1Entered via CALL to GC namespace entry (requires E perm)
2M-Elevation ON (atomic Turing abstraction)
SSCAN — Mark all CR-referenced entries as LIVE via G-bit
Walk CR0–CR15, for each non-null GT: markLive(GT.index)
Walk nsClistMap: for each c-list child, markLive(childIdx)
IIDENTIFY — Entries with G-bit matching garbage polarity are garbage
G-bit == gcPolarity → LIVE; G-bit != gcPolarity → GARBAGE
CCLEAR — Zero out garbage entries, bump their version (stale GT invalidation)
Zeroed entries become available for reuse; version bump kills stale GTs
FFLIP — Toggle gcPolarity (bidirectional G-bit)
Next cycle: LIVE = opposite polarity, no re-mark needed
3M-Elevation OFF
4RETURN — exit atomic abstraction

FAULT Handler

All validation failures across both gates route to a single FAULT handler:
NULL_CAP — null GT where non-null required
BOUNDS — index out of range or null entry
VERSION — GT version ≠ namespace entry version
SEAL — FNV seal validation failure
PERMISSION — GT lacks required permission bit
BIND — B=0 on source GT (mSave rejects)
FAR — F=1 on target (foreign/remote object, needs HTTP/tunnel)
CR_OCCUPIED — writing to non-empty CR
TPERM_RSV — reserved TPERM preset code
INVALID_OP — unknown opcode
Effect: HALT machine, log fault (type, message, PC, step count), emit fault event
