# Lump Security — Trust Chain and Forgery Detection

Church Machine · lump integrity from compiler to runtime

---

## Implementation Phases

```
Phase 1 (NOW):    header valid? → issue GTs with CRC-16/CCITT
Phase 2 (LATER):  SHA256(zip) matches URL? → then Phase 1
Phase 3 (LATER):  signature valid? → then Phase 2
```

Each phase adds one guard at the front of the same pipeline.
Nothing in an earlier phase needs to change when a later phase is added.

---

## Phase 1 — Current Target

### What the compiler produces

| Requirement | Reason |
|---|---|
| Header word correct (magic 0x1F, n-6, cc, typ) | Mint rejects malformed headers |
| Freespace region is all-zero | Required for Phase 2 deterministic SHA256 when added |
| Code entry at word 1 | Hardware always sets PC=1 on CALL |
| Instruction layout is deterministic | Required for Phase 2 reproducible hash when added |

The compiler produces raw lump words and an import table. It does not compute
CRCs, GTs, hashes, or signatures — those are all Mint and IDE responsibilities.

### What Mint does (Phase 1)

```
1. Receive lump binary (local install — no URL, no zip yet)
2. Validate header word: magic=0x1F, sane n-6, cc, typ=0x0
3. Assign object_id and gt_seq from pool
4. Build X-GT and L-GT CRs, compute CRC-16/CCITT
5. Write Object NS slot
6. Return GTs and 3-word Lump NS slot entry
```

### What Mint does NOT compute

| Item | Who computes it | When |
|---|---|---|
| CRC-16/CCITT on the GT | Mint | Phase 1 — now |
| SHA256 of the zip | IDE | Phase 2 — deferred |
| Code signature | Developer / IDE | Phase 3 — deferred |

## Phase 2 — Deferred: Content Addressing

When network distribution is added, the IDE packages the lump into a zip and
computes a SHA256 hash that becomes the content address in the `cm://` URL.

### IDE packaging (Phase 2)

```
1. Run compiler  → lump binary (raw words, already zero-filled)
2. Bundle into zip:
     <Label>@<ver>.bin          ← lump binary
     <Label>@<ver>.imports.json ← import table (slot → cm:// URL)
   Zip metadata: timestamps zeroed for reproducible SHA256
3. Compute SHA256(zip)          ← this becomes the content address
4. Form cm:// URL:
     cm://<domain>/<Label>@sha256:<hex>/v<ver>
```

### Mint additions (Phase 2)

```
0. Fetch zip from cm:// URL
1. Verify SHA256(zip) matches hash in URL    ← NEW: catches transit tampering
2. Inflate zip
   ... then Phase 1 steps 2–6 unchanged
```

## Phase 3 — Deferred: Code Signing

Publisher identity. Mint refuses GTs for any lump whose signature does not
verify against an approved key list.

### IDE additions (Phase 3)

```
5. Sign SHA256(zip) with developer private key → signature bytes
6. Bundle signature + public key certificate alongside zip
   (convention TBD: zip comment field or sidecar .sig file)
7. Publish zip + signature to network or local namespace
```

### Mint additions (Phase 3)

```
0. Fetch zip from cm:// URL
1. Verify SHA256(zip) matches hash in URL        (Phase 2)
2. Verify signature against approved key list    ← NEW: publisher identity
3. Inflate zip
   ... then Phase 1 steps 2–6 unchanged
```

## Damage Containment — All Phases

Regardless of which phase is active, a malicious lump is structurally sandboxed.
Code signing bounds *who can install* a lump. Architecture bounds *what it can do*.

| Attempted attack | Why it fails |
|---|---|
| Read another abstraction's data | No GT for that region — lump only holds GTs it was explicitly given |
| Write into another abstraction's c-list | No L/S-rights GT for the target — DWRITE cannot reach it |
| Forge a GT by writing raw bytes | Only Mint issues GTs — raw bytes are never interpreted as capabilities |
| Escalate permissions after TPERM | TPERM only clears bits; B exclusion from CRC does not allow re-addition |
| Call a privileged abstraction | Requires an E-GT the lump was not given |
| Replay a revoked GT | gt_seq mismatch → LOAD nullifies slot and faults |

## Impact Summary

| Component | Phase 1 (now) | Phase 2 (later) | Phase 3 (later) |
|---|---|---|---|
| Compiler | Correct header word; zero-fill freespace | No change | No change |
| IDE | Run compiler; local install | Zip packaging; zeroed timestamps; SHA256 URL | Sign hash; publish signature |
| Mint | Header validation; issue GTs with CRC | SHA256 URL verification | Signature verification |
| Hardware / LOAD | CRC check; gt_seq revocation | No change | No change |
